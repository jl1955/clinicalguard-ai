Add structured trace emission to the governance pipeline interceptor

8/8 sessions (100%)

PROBLEM: The ClinicalGuard AI agent produces zero trace output. The evaluation framework requires per-simulation trace files containing structured records of each tool call (scan_for_phi, check_gcp_compliance, compute_final_status, forward_to_llm, insert_log) with their inputs and outputs. All 8 simulations fail with 'No trace source found for simulation' because the traces/ directory is empty. The agent's process_query function in /agent/proxy/interceptor.py correctly orchestrates all 5 tools in the right order (parallel PHI+compliance scan → decision engine → conditional LLM forwarding → audit logging), but none of these invocations produce trace records. Every grader check across all 8 categories (phi_detection, risk_scoring, compliance_checking, governance_decision, llm_forwarding_control, pipeline_execution, response_integrity, data_accuracy) uses {{ trace }} template variables that resolve to empty data, causing all evaluations to fail. SOLUTION: Add structured trace logging to /agent/proxy/interceptor.py's `process_query` function. Before and after each tool call, emit a JSONL trace entry to stdout (or a configurable trace file path) with the following structure: {"tool": "<tool_name>", "type": "call"|"result", "input": <serialized_input>, "output": <serialized_output>, "timestamp": "<ISO8601>"}. Specifically: (1) After `scan_for_phi(prompt)` returns, log {tool: 'scan_for_phi', input: {prompt: prompt}, output: phi_result}. (2) After `check_gcp_compliance(prompt)` returns, log {tool: 'check_gcp_compliance', input: {prompt: prompt}, output: compliance_result}. (3) After `compute_final_status(phi_result, compliance_result)` returns, log {tool: 'compute_final_status', input: {phi_result, compliance_result}, output: decision}. (4) After `forward_to_llm(llm_target, prompt)` returns (if called), log {tool: 'forward_to_llm', input: {llm_target, prompt}, output: llm_response}. (5) After `insert_log(data)` returns, log {tool: 'insert_log', input: data, output: {row_id}}. (6) At the end, log the full API response payload. Use Python's json module to serialize and print each line to stdout, or write to a trace file whose path is derived from the session_id (e.g., f'traces/trace_{session_id}.jsonl'). Also add the user's original request as the first trace entry: {tool: 'proxy_query', type: 'request', input: {user_id, session_id, llm_target, prompt}}.
